# Go 代码规范

本文档规定灵畿 CLI 的 Go 代码编写规范，包括代码风格、错误处理、日志规范等。

## 代码风格

### 格式化

- 使用 `gofmt` 格式化代码
- 使用 `goimports` 管理 import
- 行长度不超过 120 字符
- 使用 tab 缩进（宽度 4）

### 命名规范

#### 包名

- 使用小写单词，无下划线
- 简短但有描述性
- 避免使用复数

```go
// 好的命名
package api
package config

// 不好的命名
package api_client    // 使用下划线
package configs       // 使用复数
```

#### 变量名

- 驼峰命名法（首字母小写）
- 简短但清晰
- 避免单字母（循环变量除外）

```go
// 好的命名
workspaceKey := "XXJSxiaobaice"
userCount := 10

// 不好的命名
wk := "XXJSxiaobaice"     // 过于简短
workspace_key := ""       // 使用下划线
```

#### 常量名

- 使用驼峰命名法
- 或全大写+下划线（仅用于导出的常量）

```go
const (
    DefaultTimeout = 30 * time.Second      // 驼峰命名
    DEFAULT_RETRY_COUNT = 3               // 全大写（导出）
)
```

#### 结构体名

- 驼峰命名法
- 首字母大写表示导出

```go
type CommandContext struct {
    Config *config.Config
    Client *api.Client
}

type commandResult struct {  // 未导出
    success bool
}
```

#### 接口名

- 以 `er` 结尾（表示执行者）
- 或描述能力的名称

```go
type Reader interface {
    Read(p []byte) (n int, err error)
}

type StringWriter interface {
    WriteString(s string) (int, error)
}
```

#### 方法名

- 驼峰命名法
- 导出方法首字母大写
- 避免使用 Get/Set 前缀（除非必要）

```go
func (c *CommandContext) Headers() map[string]string    // 好的命名
func (c *CommandContext) GetHeaders() map[string]string // 避免不必要的 Get 前缀
```

### 注释规范

#### 包注释

每个包都应有包注释，位于 package 声明之前：

```go
// Package api provides HTTP client and API wrappers for the Lingji platform.
// It handles authentication, request building, and response parsing.
package api
```

#### 函数/方法注释

- 以函数名开头
- 说明功能、参数、返回值
- 使用完整句子

```go
// NewCommandContext creates a new command context with all dependencies initialized.
// It handles config loading, logger setup, and HTTP client creation.
//
// Usage:
//
//     ctx, err := common.NewCommandContext(debugMode, insecureSkipVerify, dryRun, cookie)
//     if err != nil {
//         fmt.Fprintf(os.Stderr, "Error: %v\n", err)
//         os.Exit(1)
//     }
//     defer ctx.Close()
func NewCommandContext(debugMode, insecure bool, dryRun bool, cookie string) (*CommandContext, error) {
    // ...
}
```

#### 结构体注释

```go
// CommandContext holds all dependencies needed for command execution.
// It provides a unified way to initialize config, HTTP client, and logger.
type CommandContext struct {
    // Config stores the loaded configuration.
    Config *config.Config

    // Client is the HTTP client for API calls.
    Client *api.Client
}
```

#### 特殊注释

```go
// TODO: 实现错误重试机制
// FIXME: 这里有一个已知的并发问题
// NOTE: 这个函数不适用于批量操作
// Deprecated: Use NewClient instead.
```

## 错误处理

### 错误包装

使用 `fmt.Errorf` 的 `%w` 动词包装错误：

```go
result, err := api.CreateRequirement(ctx, req)
if err != nil {
    return nil, fmt.Errorf("创建需求失败: %w", err)
}
```

### 错误检查

```go
// 立即检查错误
result, err := someFunction()
if err != nil {
    return err
}
// 使用 result

// 避免
result, err := someFunction()
if err == nil {
    // 处理成功
}
```

### 自定义错误类型

```go
type APIError struct {
    Code    string
    Message string
    Status  int
}

func (e *APIError) Error() string {
    return fmt.Sprintf("API error %s: %s", e.Code, e.Message)
}

// 使用 errors.As 检查错误类型
var apiErr *APIError
if errors.As(err, &apiErr) {
    // 处理 API 错误
}
```

### 哨兵错误

```go
var ErrNotFound = errors.New("resource not found")
var ErrUnauthorized = errors.New("unauthorized")

// 使用 errors.Is 检查
if errors.Is(err, ErrNotFound) {
    // 处理未找到
}
```

## 日志规范

### 使用 Zap 日志库

```go
import "go.uber.org/zap"

// 在 CommandContext 中使用
func (c *CommandContext) Debug(msg string, fields ...zap.Field) {
    if c.DebugMode && c.Logger != nil {
        c.Logger.Debug(msg, fields...)
    }
}
```

### 日志级别

| 级别 | 使用场景 |
|------|----------|
| Debug | 详细的调试信息，只在调试模式输出 |
| Info | 重要的业务事件 |
| Warn | 警告，需要关注但不一定处理 |
| Error | 错误，需要处理 |
| Fatal | 致命错误，程序退出（避免使用） |

### 结构化日志

```go
// 好的写法
logger.Debug("Creating requirement",
    zap.String("name", reqName),
    zap.String("workspace", workspaceKey),
    zap.Int("priority", priority),
)

// 不好的写法
logger.Debug(fmt.Sprintf("Creating requirement %s in %s", reqName, workspaceKey))
```

### 日志字段命名

```go
// 使用驼峰命名
zap.String("workspaceKey", workspaceKey)
zap.Int("gitProjectId", gitProjectID)
zap.Duration("executionTime", elapsed)

// 布尔值使用 is/has 前缀
zap.Bool("isSuccess", success)
zap.Bool("hasMoreData", hasMore)
```

## 并发规范

### goroutine

- 必须有退出机制
- 使用 `sync.WaitGroup` 等待
- 避免 goroutine 泄露

```go
func processItems(items []Item) {
    var wg sync.WaitGroup
    semaphore := make(chan struct{}, 10) // 限制并发数

    for _, item := range items {
        wg.Add(1)
        semaphore <- struct{}{} // 获取信号量

        go func(i Item) {
            defer wg.Done()
            defer func() { <-semaphore }() // 释放信号量

            process(i)
        }(item)
    }

    wg.Wait()
}
```

### 互斥锁

```go
type Cache struct {
    mu    sync.RWMutex
    items map[string]Item
}

func (c *Cache) Get(key string) (Item, bool) {
    c.mu.RLock()
    defer c.mu.RUnlock()
    item, ok := c.items[key]
    return item, ok
}

func (c *Cache) Set(key string, item Item) {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.items[key] = item
}
```

### 通道规范

```go
// 有缓冲 vs 无缓冲
tasks := make(chan Task)       // 无缓冲，同步
tasks := make(chan Task, 100)  // 有缓冲，异步

// 只读/只写通道
func producer(ch chan<- Item)   // 只写
func consumer(ch <-chan Item)   // 只读

// 关闭通道
close(ch)

// 检查通道关闭
val, ok := <-ch
if !ok {
    // 通道已关闭
}
```

## 测试规范

### 测试命名

```go
// 测试函数
func TestCreateRequirement(t *testing.T)
func TestCreateRequirement_WithYAML(t *testing.T)
func TestCreateRequirement_InvalidInput(t *testing.T)

// 基准测试
func BenchmarkParseYAML(b *testing.B)

// 示例测试
func ExampleCommand()
```

### 测试结构

```go
func TestFeature(t *testing.T) {
    // 准备
    setup()
    defer teardown()

    // 执行
    result, err := function()

    // 验证
    if err != nil {
        t.Fatalf("unexpected error: %v", err)
    }
    if result != expected {
        t.Errorf("got %v, want %v", result, expected)
    }
}
```

### 表驱动测试

```go
func TestCalculate(t *testing.T) {
    tests := []struct {
        name     string
        input    int
        expected int
        wantErr  bool
    }{
        {"positive", 5, 10, false},
        {"zero", 0, 0, false},
        {"negative", -1, 0, true},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            result, err := Calculate(tt.input)
            if (err != nil) != tt.wantErr {
                t.Errorf("Calculate() error = %v, wantErr %v", err, tt.wantErr)
                return
            }
            if result != tt.expected {
                t.Errorf("Calculate() = %v, want %v", result, tt.expected)
            }
        })
    }
}
```

### 辅助函数

```go
// 标记为测试辅助函数
func setupTestData(t *testing.T) {
    t.Helper()
    // ...
}
```

## 性能规范

### 避免不必要的分配

```go
// 不好：每次迭代都分配
for i := 0; i < n; i++ {
    s := string(buffer)  // 分配
    process(s)
}

// 好：复用变量
var s string
for i := 0; i < n; i++ {
    s = string(buffer)
    process(s)
}
```

### 字符串拼接

```go
// 少量字符串：使用 +
msg := "Hello, " + name

// 大量字符串：使用 strings.Builder
var b strings.Builder
b.Grow(1000)  // 预分配
for _, s := range items {
    b.WriteString(s)
}
result := b.String()
```

### 预分配切片/map

```go
// 已知大小
items := make([]Item, 0, len(source))
for _, s := range source {
    items = append(items, process(s))
}

// map
m := make(map[string]int, 100)
```

```bash
golangci-lint run
```
