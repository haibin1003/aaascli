01 / 项目背景

为什么我们要做这款工具？

平台操作依赖浏览器

山东能力开放平台所有查询、订购、授权操作都必须在网页端完成，AI 助手无法直接调用，只能「口头指导」用户手动点击。

人工查询效率低下

平台汇聚了 325+ 业务能力和 3800+ 具体 API 服务，纯人工浏览查找耗时久，难以快速匹配业务需求。

AI 价值无法落地

AI 拥有强大的信息整合和代码生成能力，但缺乏直接访问企业内部平台的通道，导致「有脑子、没手脚」。

💡

核心解决思路

为 AI 助手打造一款专用的命令行客户端（CLI），通过逆向平台接口协议，让 AI 能够直接查询能力目录、检索 API 服务、查看接口详情，甚至基于返回数据生成调用代码和业务方案。

03 / 核心功能

四大模块，覆盖能力查询、服务检索、应用管理和知识赋能

📦

能力管理（Ability）

  • 查询全部 325+ 业务能力列表
  • 按关键词快速搜索能力
  • 查看能力详情（描述、提供方、分类）
  • 浏览能力下挂载的服务菜单
  • 查询我的已订购能力
🔌

数字服务管理（Service）

  • 查询全量 3800+ 服务目录
  • 搜索具体 API 服务接口
  • 查看服务详情：URL、方法、协议
  • 获取请求/响应示例 JSON
  • 查看接口负责人和联系方式
📚

知识库（Knowledge）

  • 内置能力使用指南（Markdown）
  • 典型业务场景和组合方案
  • Python / Java / Go 代码模板
  • 常见问题与故障排查
  • 支持全文搜索和片段提取
🔐

认证与辅助（Auth & Helper）

  • Chrome 插件一键提取 Token
  • CLI 自动保存登录凭证
  • TLS 跳过验证，适配内网环境
  • AI onboarding 引导指南
  • Pretty-print JSON 输出

04 / 使用流程

四步完成部署，AI 即可开始服务

1

AI 安装 CLI

根据系统架构选择二进制文件
放入 PATH 即可运行

2

用户导入插件

Chrome 加载 sdp-login-helper
登录平台后复制 Token 给 AI

3

AI 执行登录

运行 sdp login 完成认证
即可查询能力、生成方案

4

持续智能服务

查询接口、搜索知识库
输出方案和调用代码

// 典型交互示例：搜索能力 -> 查看详情 -> 生成代码
$ sdp ability search "短信定位" --size 5
$ sdp ability view CA202512091059315001040209490536
$ sdp service view SE202604081915467601182242
$ sdp knowledge search "代码示例"

05 / 成效与价值

我们做这件事，究竟带来了什么改变？

🚀 AI 从「顾问」变成「执行者」

过去 AI 只能告诉用户「你可以去平台搜一下这个能力」。现在 AI 可以直接执行查询、分析返回结果、输出方案，真正实现了「动手」而不仅是「动嘴」。

核心价值

⏱️ 能力匹配效率提升 10 倍以上

平台 325+ 能力、3800+ 服务全靠人工浏览需要数小时。通过 CLI + AI 检索，用户只需描述业务场景，30 秒内即可获得精准的能力推荐和接口详情。

效率提升

🛠️ 降低技术接入门槛

AI 可基于接口返回的 requestExample 和 responseExample，自动生成 Python / Java / Go 调用代码。普通业务人员无需阅读复杂文档即可完成对接。

降本增效

🌐 一次构建，全平台覆盖

同时提供 Windows(x64/ARM64)、Linux(x64/ARM64)、macOS(Intel/Apple Silicon) 六个版本的二进制文件，无论 AI 运行在哪种环境都能即装即用。

工程优势

📚 知识沉淀，越用越聪明

内置知识库将短信触达、机器视觉、定位服务等典型方案固化到二进制中。AI 可以直接引用最佳实践，而不是每次从零推理，回答质量更稳定。

知识资产

🔒 安全合规，不离线不泄露

所有依赖均已 Vendor 化，可在完全离线的内网环境编译和运行。Token 和 Cookie 仅保存在本地配置文件，不经过任何第三方服务。

安全可控
325+ 业务能力已接入查询
3800+ 数字服务/API 可检索
6 平台二进制覆盖
5 内置知识文档
0 运行时外部依赖
