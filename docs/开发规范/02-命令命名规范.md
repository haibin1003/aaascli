# 命令命名规范

本文档规定灵畿 CLI 的命令、子命令、标志的命名规范，确保一致性和可预测性。

## 命名原则

1. **简洁性**：使用简短、易记的单词
2. **一致性**：相同概念使用相同术语
3. **可预测性**：命名应符合用户直觉
4. **语义化**：命名应准确表达功能含义

## 命令命名

### 一级命令（父命令）

使用 **名词** 或 **动名词**，表示操作对象：

| 命令 | 说明 | 示例 |
|------|------|------|
| `req` | 需求管理（requirement 缩写） | `lc req list` |
| `bug` | 缺陷管理 | `lc bug create` |
| `task` | 任务管理 | `lc task create` |
| `pr` | 合并请求（Pull Request） | `lc pr review` |
| `repo` | 代码仓库 | `lc repo create` |
| `space` | 研发空间 | `lc space list` |
| `lib` | 文档库（library 缩写） | `lc lib list` |

**命名规则**：
- 优先使用行业通用缩写（`pr`, `repo`, `lib`）
- 避免使用复数形式（用 `req` 而非 `reqs`）
- 保持与灵畿平台术语一致

### 二级命令（子命令）

使用 **动词** 或 **动词短语**，表示操作动作：

| 子命令 | 用途 | 示例 |
|--------|------|------|
| `list` | 查询列表 | `lc req list` |
| `create` | 创建 | `lc req create` |
| `view` | 查看详情 | `lc req view <id>` |
| `update` | 更新 | `lc req update <id>` |
| `delete` | 删除 | `lc req delete <id>` |
| `search` | 搜索 | `lc req search <keyword>` |
| `review` | 审核 | `lc pr review <id>` |
| `merge` | 合并 | `lc pr merge <id>` |

**命名规则**：
- 使用动词原形（`create` 而非 `creates`）
- 避免使用 `get`，使用 `view` 或 `list`
- 避免使用 `remove`，使用 `delete`

### 三级命令

用于复杂命令的子分类：

```
lc space project list      # space → project → list
lc skills hub search       # skills → hub → search
lc lib folder create       # lib → folder → create
```

**命名规则**：
- 三级命令前使用名词分类
- 保持层级清晰，避免过深的嵌套（最多 3 层）

## 标志命名

### 短标志（单字符）

| 标志 | 含义 | 示例 |
|------|------|------|
| `-w` | workspace | `-w XXJSxiaobaice` |
| `-k` | insecure（skip verify） | `-k` |
| `-d` | debug | `-d` |
| `-t` | title | `-t "需求标题"` |
| `-D` | description | `-D "描述内容"` |
| `-s` | source | `-s feature-branch` |
| `-m` | main/master | `-m master` |
| `-f` | file | `-f config.yaml` |
| `-l` | limit | `-l 10` |
| `-p` | page/project | `-p 1` / `-p R24113J3C04` |

## 分页参数规范

所有支持分页查询的命令**必须**使用统一的分页参数：

| 参数 | 短标志 | 长标志 | 默认值 | 说明 |
|------|--------|--------|--------|------|
| 页码 | `-p` | `--page` | `1` | 从 1 开始计数 |
| 每页数量 | `-l` | `--limit` | `10` | 每页返回的数据条数 |

### 使用示例

```bash
# 查询第 1 页，每页 10 条
lc req list -p 1 -l 10

# 查询第 2 页，每页 20 条
lc repo list -p 2 -l 20

# 手动参数模式
lc bug list --workspace-key XXJSxiaobaice -p 1 -l 50
```

### 实现规范

在 Cobra 命令中，使用 `IntVarP` 同时支持短标志和长标志：

```go
// 正确的实现方式
listCmd.Flags().IntVarP(&page, "page", "p", 1, "页码")
listCmd.Flags().IntVarP(&limit, "limit", "l", 10, "每页数量")

// 错误的实现方式 - 缺少短标志
listCmd.Flags().IntVar(&page, "page", 1, "页码")
listCmd.Flags().IntVar(&limit, "size", 10, "每页数量")
```

### 已统一的分页命令

以下命令已统一使用 `-p, --page` 和 `-l, --limit`：

- `lc repo list` / `lc repo search`
- `lc req list`
- `lc task list`
- `lc pr list`
- `lc bug list`

**命名规则**：
- 单个小写字母
- 常用概念保持跨命令一致
- 避免使用容易混淆的字母（`o`, `O`, `0`）

### 长标志（多字符）

| 标志 | 含义 | 示例 |
|------|------|------|
| `--workspace-key` | 研发空间 Key | `--workspace-key XXJSxiaobaice` |
| `--project-code` | 项目代码 | `--project-code R24113J3C04` |
| `--git-project-id` | Git 项目 ID | `--git-project-id 44142` |
| `--dry-run` | 模拟执行 | `--dry-run` |
| `--pretty` | 格式化 JSON | `--pretty` |
| `--template-simple` | 简洁模板 | `--template-simple` |

**命名规则**：
- 使用小写字母，多个单词用连字符 `-` 连接
- 优先使用名词，布尔标志使用形容词
- 与灵畿平台 API 字段名保持一致

### 布尔标志命名

布尔标志使用 **形容词** 或 **助动词**：

```go
// 好的命名
--template-simple      // 简洁模板
--insecure             // 不安全模式（跳过 SSL）
--dry-run             // 模拟运行

// 不好的命名
--use-simple-template  // 冗长
--skip-ssl-verify     // 与 --insecure 重复
```

**特殊命名**：`--template-simple`

- 使用 `template-` 前缀区分不同模板类型
- 后续可扩展：`--template-full`, `--template-minimal`

## 参数命名

### 位置参数

位置参数使用 **小写驼峰命名法**：

```go
// 示例：lc req view <objectId>
Args: cobra.ExactArgs(1),
Run: func(cmd *cobra.Command, args []string) {
    objectID := args[0]  // 使用驼峰命名
}
```

**命名规则**：
- 单数形式（`objectID` 而非 `objectIDs`）
- ID 类参数统一使用 `xxxID` 格式（大写 ID）
- 名称类参数使用 `xxxName` 或 `xxxKey`

### 参数顺序

多个位置参数应按重要性排序：

```go
// 示例：lc task create <reqObjectId> <taskName>
Args: cobra.ExactArgs(2),
Run: func(cmd *cobra.Command, args []string) {
    reqObjectID := args[0]  // 父对象 ID 在前
    taskName := args[1]     // 新建对象名称在后
}
```

## 命名示例

### 完整命令示例

```bash
# 创建需求
lc req create "需求标题" \
  --workspace-key XXJSxiaobaice \
  --project-code R24113J3C04 \
  --template-simple

# 查询需求列表
lc req list \
  --workspace-key XXJSxiaobaice \
  --limit 10

# 创建合并请求
lc pr create \
  --title "修复登录bug" \
  --source feature-branch \
  --target master \
  --git-project-id 44142 \
  --workspace-key XXJSxiaobaice

# 审核合并请求
lc pr review 123 \
  --git-project-id 44142 \
  --workspace-key XXJSxiaobaice \
  --type approve
```

### Go 代码中的命名

```go
// 命令定义
var reqCreateCmd = &cobra.Command{
    Use:   "create [name]",
    Short: "创建需求",
    Long:  `根据提供的名称和参数创建新需求`,
    Run: func(cmd *cobra.Command, args []string) {
        // 参数获取
        workspaceKey, _ := cmd.Flags().GetString("workspace-key")
        projectCode, _ := cmd.Flags().GetString("project-code")
        useSimpleTemplate, _ := cmd.Flags().GetBool("template-simple")

        // 业务逻辑...
    },
}

// 初始化标志
func init() {
    reqCreateCmd.Flags().StringP("workspace-key", "w", "", "研发空间 Key")
    reqCreateCmd.Flags().String("workspace-name", "", "研发空间名称")
    reqCreateCmd.Flags().StringP("project-code", "p", "", "项目代码")
    reqCreateCmd.Flags().Bool("template-simple", false, "使用简洁模板")
}
```

## 注意事项

1. **向后兼容**：已发布的命令和标志不能修改，只能新增
2. **别名支持**：可以为命令添加别名，但需谨慎使用
3. **废弃标记**：废弃的标志应标记为 `Deprecated`，并在帮助中说明

```go
// 标记废弃标志
reqCreateCmd.Flags().MarkDeprecated("old-flag", "请使用 --new-flag 替代")
```
