# 只读模式临时关闭功能需求

## 变更记录表

| 版本 | 日期 | 变更内容 | 作者 |
|------|------|----------|------|
| v1.0 | 2026-03-17 | 初始版本 | AI Assistant |

---

## 1. 需求背景

LC CLI 已实现只读模式功能，默认开启以防止误操作。但用户在某些场景下需要临时执行写入操作，如：
- 批量创建需求/任务
- 紧急修复缺陷
- 临时上传文档

当前需要手动关闭只读模式，操作完成后手动开启，容易遗忘恢复，存在安全风险。

## 2. 需求目标

为只读模式增加**临时关闭**功能，允许用户指定时长（如 10m、30m、1h），时长到期后自动恢复只读模式。

## 3. 功能需求

### 3.1 命令格式

```bash
# 临时关闭只读模式（默认时长，如 30 分钟）
lc readonly off --duration 30m

# 临时关闭 1 小时
lc readonly off --duration 1h

# 临时关闭 10 分钟
lc readonly off --duration 10m

# 永久关闭（原有功能）
lc readonly off
```

### 3.2 参数说明

| 参数 | 短格式 | 必填 | 默认值 | 说明 |
|------|--------|------|--------|------|
| --duration | -d | 否 | 无 | 临时关闭时长，格式如 10m、30m、1h、2h |

### 3.3 时长格式

支持以下格式：
- `Xm` - 分钟，如 `10m`、`30m`
- `Xh` - 小时，如 `1h`、`2h`
- 最大支持 `24h`
- 最小 `1m`

### 3.4 状态显示

临时关闭期间，查看状态应显示：

```json
{
  "success": true,
  "data": {
    "readonly": false,
    "temporary": true,
    "expireAt": "2026-03-17T12:30:00Z",
    "remainingMinutes": 25,
    "description": "当前处于临时读写模式，将在 25 分钟后自动恢复只读",
    "warning": "请谨慎执行删除和修改操作，数据变更将直接生效"
  }
}
```

### 3.5 自动恢复机制

- 每次执行写入命令前，检查是否有过期的临时关闭
- 如果过期，自动恢复只读模式并拒绝当前写入操作
- 返回提示信息告知用户已自动恢复

## 4. 非功能需求

### 4.1 安全性

- 临时关闭期间仍记录所有写入操作日志（如已有）
- 最大时长限制 24 小时，防止设置过长导致遗忘
- 过期后必须自动恢复，不可因程序错误保持开放

### 4.2 兼容性

- 不影响原有 `lc readonly off` 永久关闭功能
- 与现有配置格式兼容，新增字段不影响旧版本

## 5. 验收标准

| 编号 | 验收项 | 验收标准 |
|------|--------|----------|
| AC1 | 临时关闭命令 | `lc readonly off --duration 30m` 成功关闭只读模式并设置过期时间 |
| AC2 | 状态显示 | `lc readonly` 显示临时关闭状态和剩余时间 |
| AC3 | 自动恢复 | 过期后执行写入命令，自动恢复只读模式并拒绝操作 |
| AC4 | 格式验证 | 非法时长格式返回错误提示 |
| AC5 | 范围验证 | 超过 24h 或小于 1m 返回错误提示 |
| AC6 | 永久关闭 | 不带 --duration 参数仍保持原有永久关闭行为 |
