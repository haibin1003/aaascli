## 常用工作流

### 1. 需求开发流程

```bash
# 1. 创建需求
lc req create "新功能需求" -w XXJSxiaobaice --project-code R24113J3C04

# 2. 分解任务（使用返回的需求 objectId）
lc task create nCKhZGcBKu "实现登录接口" -w XXJSxiaobaice --project-code R24113J3C04

# 3. 创建 MR
lc pr create -t "实现登录功能" -s feature/login -m master \
  --git-project-id 44142 -w XXJSxiaobaice

# 4. 审核 MR
lc pr review 123 --git-project-id 44142 -w XXJSxiaobaice --type approve

# 5. 合并 MR
lc pr merge 123 --git-project-id 44142 -w XXJSxiaobaice
```

### 2. Bug 修复流程

```bash
# 1. 创建 Bug
lc bug create -t "登录按钮无法点击" -D "点击无响应" -p R24113J3C04 \
  -w XXJSxiaobaice --template-simple

# 2. 修复后更新状态（假设状态 ID 为 1642790711574196231）
lc bug update-status 2030505333202456578 1642790711574196231 -w XXJSxiaobaice
```
