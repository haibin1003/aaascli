# E2E 测试策略

## 背景

随着自动探测功能的完善，现在命令支持两种使用模式：

1. **手动参数模式** - 用户显式提供所有参数（如 `--workspace-key`, `--git-project-id`）
2. **自动探测模式** - 在 Git 仓库目录下执行，自动从 `.git/config` 解析参数

这两种模式的输入参数差异很大，需要分别测试。

---

## 测试分类

### 1. 手动参数模式测试 (`e2e/integration/manual/`)

**目标**: 验证用户显式提供参数时命令的正确性

**使用研发空间**: **小白测研发项目** (`XXJSxiaobaice`)
- 手动模式必须在小林测空间执行测试
- 所有参数显式指定，不依赖自动探测

**特点**:
- 所有必需参数都通过命令行显式传递
- 不依赖 Git 仓库上下文
- 可在任何目录下执行
- 适合 CI/CD 环境

**测试重点**:
- 参数解析正确性
- 参数组合验证
- 错误参数处理
- 权限验证

**示例命令**:
```bash
lc req create "需求名称" --workspace-key XXJSxiaobaice --project-code R24113J3C04
lc repo list --workspace-key XXJSxiaobaice
```

### 2. 自动探测模式测试 (`e2e/integration/autodetect/`)

**目标**: 验证在 Git 仓库目录下自动探测参数的功能

**使用研发空间**: **小白测研发项目** (`XXJSxiaobaice`) - 通过 Git 自动探测
- 测试使用小白测空间的 forever 仓库 URL
- 参数通过 `lc detect` 自动探测，无需显式指定
- 与手动模式在同一空间，便于统一管理测试数据

**特点**:
- 在 Git 仓库目录下执行
- 参数从 Git remote URL 自动解析
- 需要预先克隆仓库或设置 Git 环境
- 更贴近实际开发场景

**测试重点**:
- 自动探测正确性
- Git 仓库识别
- 参数推断逻辑
- 探测失败处理

**示例命令**:
```bash
cd /path/to/lc/repo
lc req create "需求名称"  # 自动探测 workspace-key 和 project-code
lc repo list              # 自动探测 workspace-key
```

## 测试基类设计

### 手动模式测试基类

```go
// ManualModeTest 手动参数模式测试基类（使用小白测空间）
type ManualModeTest struct {
    T             *testing.T
    CLI           *CLI
    Config        *TestConfig
}

func NewManualModeTest(t *testing.T) *ManualModeTest {
    return &ManualModeTest{
        T:      t,
        CLI:    NewCLI(t),
        Config: GetManualModeConfig(),  // 小白测空间配置
    }
}

func (m *ManualModeTest) Run(args ...string) *Result {
    // 手动模式默认添加 -k 和 --workspace-key（小白测）
    baseArgs := []string{"-k", "--workspace-key", m.Config.WorkspaceKey}
    return m.CLI.Run(append(args, baseArgs...)...)
}
```

### 自动探测模式测试基类

```go
// AutoDetectTest 自动探测模式测试基类（使用小白测空间）
type AutoDetectTest struct {
    T       *testing.T
    CLI     *CLI
    RepoDir string  // Git 仓库目录
    Config  *TestConfig
}

func NewAutoDetectTest(t *testing.T) *AutoDetectTest {
    repoDir := PrepareTestGitRepo(t) // 准备小白测空间的 Git 仓库
    return &AutoDetectTest{
        T:       t,
        CLI:     NewCLI(t),
        RepoDir: repoDir,
        Config:  GetAutoDetectConfig(),  // 小白测空间配置（用于验证）
    }
}

func (a *AutoDetectTest) RunInRepo(args ...string) *Result {
    // 自动探测模式在 Git 仓库目录下执行，不添加 workspace-key
    // 通过 Git remote URL 自动探测 XXJSxiaobaice 空间
    return a.CLI.RunInDir(a.RepoDir, args...)
}

func PrepareTestGitRepo(t *testing.T) string {
    // 创建临时 Git 仓库，设置 remote URL 为小白测空间仓库
    // 远程仓库: http://code-xxjs.rdcloud.4c.hq.cmcc/osc/XXJS/weibaohui-hq.cmcc/forever.git
    // 自动探测结果: workspaceKey=XXJSxiaobaice, workspaceName=小白测研发项目
}
```

## CI/CD 配置建议

```yaml
# .github/workflows/e2e.yml
jobs:
  manual-mode-test:
    name: 手动参数模式测试（小白测空间）
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run manual mode tests
        run: go test ./e2e/integration/manual/... -v
        env:
          # 小白测研发项目配置
          LC_MANUAL_WORKSPACE_KEY: XXJSxiaobaice
          LC_MANUAL_WORKSPACE_NAME: 小白测研发项目
          LC_MANUAL_PROJECT_CODE: R24113J3C04
          LC_MANUAL_GIT_PROJECT_ID: 44618
          LC_CONFIG_JSON: ${{ secrets.LC_CONFIG_JSON }}

  autodetect-mode-test:
    name: 自动探测模式测试（小白测空间）
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Git repo
        run: |
          git config --global user.email "test@example.com"
          git config --global user.name "Test User"
      - name: Run autodetect mode tests
        run: go test ./e2e/integration/autodetect/... -v
        env:
          # 小白测空间配置（用于验证自动探测结果）
          LC_AUTODETECT_WORKSPACE_KEY: XXJSxiaobaice
          LC_AUTODETECT_WORKSPACE_NAME: 小白测研发项目
          LC_AUTODETECT_PROJECT_CODE: R24113J3C04
          LC_AUTODETECT_GIT_PROJECT_ID: 44650
          LC_TEST_REPO_URL: http://code-xxjs.rdcloud.4c.hq.cmcc/osc/XXJS/weibaohui-hq.cmcc/forever.git
          LC_CONFIG_JSON: ${{ secrets.LC_CONFIG_JSON }}
```
