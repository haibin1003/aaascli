# 禁止事项清单

**本文档列出绝对禁止的行为和代码模式。违反这些规定会导致代码被拒绝。**

---

## 架构层面禁止事项

### ❌ 禁止修改现有架构

| 禁止项 | 说明 | 正确做法 |
|--------|------|----------|
| 新增输出格式 | 不得创建新的输出格式 | 必须使用统一的 JSON 格式 |
| 修改 Execute 函数 | 不得修改 `common.Execute` 的行为 | 使用现有的执行框架 |
| 新增 HTTP 客户端创建方式 | 不得自行创建 `http.Client` | 使用 `ctx.Client` |
| 新增配置加载方式 | 不得自行读取配置文件 | 使用 `ctx.Config` |
| 修改命令注册方式 | 不得修改 Cobra 的使用方式 | 按现有模式注册命令 |

### ❌ 禁止引入新依赖

以下情况**禁止**引入新依赖：

- 已有库可以实现相同功能
- 仅为了"更好"或"更简洁"而引入
- 未经团队讨论的大型依赖

**允许引入的情况：**
- 灵畿平台 API 要求的特定协议库
- 标准库确实无法实现的特殊功能

## 代码层面禁止事项

### ❌ 禁止的输出方式

```go
// ❌ 禁止：直接打印
fmt.Println("结果:", result)
fmt.Printf("名称: %s\n", name)
log.Println("处理完成")

// ❌ 禁止：直接返回 API 原始响应
fmt.Println(string(apiResponseBody))

// ✅ 正确：使用统一输出
common.Execute(func(ctx *common.CommandContext) (interface{}, error) {
    return result, nil
}, opts)
```

### ❌ 禁止的错误处理

```go
// ❌ 禁止：panic
if err != nil {
    panic(err)
}

// ❌ 禁止：忽略错误
result, _ := someFunction()

// ❌ 禁止：直接打印错误后退出
if err != nil {
    fmt.Println("错误:", err)
    os.Exit(1)
}

// ❌ 禁止：不包装错误
return nil, err

// ✅ 正确：包装错误后返回
return nil, fmt.Errorf("操作失败: %w", err)
```

### ❌ 禁止的资源管理

```go
// ❌ 禁止：不关闭响应体
resp, err := client.Send(req)
// 直接使用 resp...

// ❌ 禁止：提前返回不关闭
if someCondition {
    return result, nil  // resp 未关闭
}
resp.Body.Close()

// ✅ 正确：使用 defer
resp, err := client.Send(req)
if err != nil {
    return nil, err
}
defer resp.Body.Close()
```

### ❌ 禁止的命名

```go
// ❌ 禁止：使用下划线
workspace_key := "xxx"

// ❌ 禁止：使用复数
reqs_list := []Requirement{}

// ❌ 禁止：过短或无意义的变量名
wk := "xxx"
a := 1
tmp := something

// ❌ 禁止：匈牙利命名法
strName := "xxx"
nCount := 10

// ✅ 正确：驼峰命名
workspaceKey := "xxx"
reqList := []Requirement{}
name := "xxx"
```

### ❌ 禁止的函数签名

```go
// ❌ 禁止：不接收 CommandContext
func DoSomething(client *api.Client, config *config.Config) (Result, error)

// ❌ 禁止：暴露内部结构
func CreateRequirementRaw(cookie string, headers map[string]string, body []byte) ([]byte, error)

// ✅ 正确：接收 CommandContext
func DoSomething(ctx *common.CommandContext, params ...) (Result, error)
```

```go
// ❌ 禁止：不使用 common.Execute

    ctx, _ := common.NewCommandContext(debugMode, insecure, dryRun, cookie)

    configFile, _ := os.ReadFile("~/.lc/config.json")
    var cfg Config
    json.Unmarshal(configFile, &cfg)
    // 错误！

// ❌ 禁止：跳过自动探测
Run: func(cmd *cobra.Command, args []string) {
    workspaceKey, _ := cmd.Flags().GetString("workspace-key")
    // 不检查是否为空，不自动探测
    common.Execute(...)

// ✅ 正确：使用标准模式
Run: func(cmd *cobra.Command, args []string) {
    // 1. 获取参数
    // 2. 自动探测（如需要）
    // 3. 使用 common.Execute
}
```

--WorkspaceKey

## API 实现禁止事项

### ❌ 禁止的 API 模式

```go
// ❌ 禁止：不校验参数
func CreateXxx(ctx *common.CommandContext, req CreateRequest) (*XxxResponse, error) {
    // 直接调用 API，不检查 req.Name 是否为空
    resp, err := ctx.Client.Send(...)
}

// ❌ 禁止：不使用 defer 关闭响应
func GetXxx(ctx *common.CommandContext, id string) (*XxxResponse, error) {
    resp, err := ctx.Client.Send(req)
    body, _ := io.ReadAll(resp.Body)
    resp.Body.Close()  // 如果前面出错，这里不会执行
}

// ❌ 禁止：返回原始字节
func GetXxxRaw(ctx *common.CommandContext, id string) ([]byte, error) {
    resp, _ := ctx.Client.Send(req)
    return io.ReadAll(resp.Body)  // 错误！
}

// ❌ 禁止：自行构建完整 URL
func ListXxx(ctx *common.CommandContext) ([]Xxx, error) {
    url := "https://rdcloud.4c.hq.cmcc/moss/web/xxx/list"  // 错误！
}

// ✅ 正确：使用配置中的基础 URL
url := fmt.Sprintf("%s/xxx/list", ctx.Config.API.BaseReqURL)
```

### ❌ 禁止的错误处理

```go
// ❌ 禁止：不检查 HTTP 状态码
resp, err := ctx.Client.Send(req)
var result Response
json.NewDecoder(resp.Body).Decode(&result)

// ❌ 禁止：返回原始错误
if err != nil {
    return nil, err  // 错误！没有上下文
}

// ❌ 禁止：使用 panic
if resp.StatusCode != 200 {
    panic("API call failed")  // 绝对禁止！
}

// ✅ 正确：检查状态码并包装错误
resp, err := ctx.Client.Send(req)
if err != nil {
    return nil, fmt.Errorf("发送请求失败: %w", err)
}
defer resp.Body.Close()

if resp.StatusCode != http.StatusOK {
    body, _ := io.ReadAll(resp.Body)
    return nil, fmt.Errorf("API 返回 HTTP %d: %s", resp.StatusCode, body)
}
```

## 安全禁止事项

### ❌ 绝对禁止

| 禁止项 | 风险 | 检测方式 |
|--------|------|----------|
| 硬编码密码/密钥 | 安全漏洞 | 代码审查 |
| 提交私钥文件 | 安全漏洞 | .gitignore |
| 记录敏感信息到日志 | 信息泄露 | 代码审查 |
| 跳过 SSL 验证作为默认 | 中间人攻击 | 标志默认值 |
| 执行用户输入的命令 | 命令注入 | 代码审查 |

```go
// ❌ 禁止：硬编码敏感信息
const API_KEY = "sk-1234567890abcdef"

// ❌ 禁止：记录敏感信息
ctx.Logger.Info("Request", zap.String("cookie", ctx.Config.Cookie))

// ❌ 禁止：执行用户输入
userInput := cmd.Flags().GetString("command")
exec.Command("sh", "-c", userInput).Run()  // 危险！

// ✅ 正确：从配置读取
apiKey := ctx.Config.API.Key

// ✅ 正确：脱敏日志
ctx.Logger.Info("Request", zap.String("cookie", "***masked***"))
```

## 代码提交禁止事项

### ❌ 禁止提交的内容

- `panic()` 调用
- `fmt.Println()` 调试语句
- `os.Exit()` 在 `common.Execute` 之外
- 注释掉的调试代码
- 硬编码的测试数据（如特定项目的ID）
- 未完成的 `TODO` 没有说明
- 二进制文件
- 个人配置文件

## 设计决策禁止事项

### ❌ 禁止的设计思维

1. **"我觉得这样可以改进"**
   - 除非有明确的痛点，否则不要修改现有工作模式

2. **"这样写更简洁"**
   - 简洁不等于好，统一性更重要

3. **"未来可能需要"**
   - 只实现当前需求，不要过度设计

4. **"其他项目都是这么做的"**
   - 遵循本项目规范，不照搬其他项目

5. **"这个库更好用"**
   - 使用项目已有依赖，不引入新依赖

---

## 违规处理

如果发现违反以上禁止事项的情况：

1. **立即标记** - 在代码审查中指出
2. **要求重写** - 必须按照规范重新实现
3. **记录违规** - 记录在案，避免重复发生

**原则：宁可在统一性上保守，也不要在创新性上冒险。**
