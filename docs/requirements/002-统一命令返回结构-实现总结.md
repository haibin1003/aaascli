# 002-统一命令返回结构-实现总结

> 关联需求：XXJSLJCLIDEV-8

## 1. 实现概述

本次实现统一了所有 LC 命令的输出格式为 JSON，确保每个命令返回结构包含 `success`、`data`、`error` 和 `meta` 字段。

---

## 2. 核心修改

### 2.1 基础结构增强 (`internal/common/executor.go`)

```go
// MetaInfo 元信息
type MetaInfo struct {
    RequestID     string `json:"requestId"`
    Timestamp     string `json:"timestamp"`
    ExecutionTime int64  `json:"executionTime,omitempty"`
    Version       string `json:"version"`
}

// ErrorDetail 错误详情
type ErrorDetail struct {
    Code       string `json:"code"`
    Message    string `json:"message"`
    Details    string `json:"details,omitempty"`
    Suggestion string `json:"suggestion,omitempty"`
}

// CommandResult 统一返回结构
type CommandResult struct {
    Success bool        `json:"success"`
    Data    interface{} `json:"data,omitempty"`
    Error   interface{} `json:"error,omitempty"`
    Meta    MetaInfo    `json:"meta"`
}
```

### 2.2 执行函数统一

- `Execute()` - 自动包装为统一 JSON 格式
- `ExecuteWithResult()` - 返回成功响应（含 meta 信息）

## 5. 破坏性变更

### 5.1 不向后兼容

- 所有命令输出现在都是 JSON 格式
- 移除了表格、文本等人类可读格式
- 交互式确认（如覆盖确认）改为直接报错

### 5.2 已知限制

- `helper extract` 不再询问是否覆盖，如果目录存在直接报错
- `onboard` 指南内容现在作为 JSON 字符串返回，需要解析后显示
