## 核心原则

### 1. 统一输出格式
所有命令必须返回统一的 JSON 输出格式：

```json
{
  "success": true,
  "data": { ... },
  "meta": {
    "timestamp": "2026-03-15T10:30:00Z",
    "executionTime": 123,
    "version": "v1.2.3"
  }
}
```

### 2. 自动探测优先
支持自动探测的命令应优先使用自动探测，减少用户输入：

```go
// 自动探测 workspace-key
autoResult := common.TryAutoDetect(true)
if autoResult.Success {
    workspaceKey = autoResult.Context.WorkspaceKey
}
```

### 3. 分页参数统一

所有分页命令使用统一的分页参数 `-p, --page` 和 `-l, --limit`：

```bash
# 正确的用法
lc req list -p 1 -l 10
lc bug list --workspace-key XXJSxiaobaice -p 2 -l 20
```

详细规范参见 [02-命令命名规范.md](./02-命令命名规范.md#分页参数规范)。

### 4. 安全设计
- E2E 测试必须在 `/tmp/` 目录下运行
- 强制使用"小白测"研发空间进行测试
- 敏感操作需要二次确认

