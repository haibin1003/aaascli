## 1. 测试范围

本文档定义 OTP 二次验证功能的测试策略、测试用例和验收标准。

## 2. 测试策略

### 2.1 测试层级

| 层级 | 类型 | 说明 |
|------|------|------|
| 单元测试 | 白盒测试 | OTP 服务算法正确性 |
| 集成测试 | 黑盒测试 | 完整 OTP 流程 |
| 兼容性测试 | 黑盒测试 | 第三方验证器兼容性 |

### 2.2 测试环境

- **开发环境**: 本地开发机
- **测试工具**: Go testing 框架
- **验证器应用**: Google Authenticator, Microsoft Authenticator
- **依赖库**: github.com/skip2/go-qrcode

## 3. 测试用例

### 3.1 OTP 服务单元测试

#### TC-001: 密钥生成
| 项目 | 内容 |
|------|------|
| 测试目的 | 验证生成的密钥符合规范 |
| 前置条件 | 无 |

| 预期结果 | 1. 密钥长度为 32 个字符 (Base32 编码的 20 字节)<br>2. 只包含大写字母 A-Z 和数字 2-7 |
| 验收标准 | 密钥格式正确，可解码 |
| 实现位置 | `internal/service/otp_service.go:GenerateSecret()` |

#### TC-002: TOTP 码生成
| 项目 | 内容 |
|------|------|
| 测试目的 | 验证 TOTP 码生成算法正确 |
| 前置条件 | 已知密钥和固定时间 |
| 测试步骤 | 1. 使用测试向量验证 `GenerateCode()` |

#### TC-003: TOTP 码验证

### 3.2 OTP 命令集成测试

#### TC-101: OTP 初始化流程（setup）
| 项目 | 内容 |
|------|------|
| 测试目的 | 验证 OTP 初始化完整流程 |
| 前置条件 | OTP 未启用 |
| 测试步骤 | 1. 执行 `lc otp setup`<br>2. 观察二维码显示（ASCII 字符）<br>3. 观察密钥显示（分组格式）<br>4. 输入正确验证码 |
| 预期结果 | 1. 显示 ASCII 二维码（使用 ▄▀█ 等字符）<br>2. 显示 10 个备用码<br>3. 显示账户名（不含乱码）<br>4. 验证通过后保存配置 |

| 验收标准 | 验证结果正确，会话正确创建 |
| 特殊说明 | 命令不受只读模式限制 |

## 4. 验收标准

### 4.1 功能验收

| 验收项 | 标准 |
|--------|------|
| OTP 初始化 | 可成功生成密钥、显示 ASCII 二维码、保存配置 |
| OTP 验证 | 正确码验证通过，错误码验证失败 |
| 会话管理 | 验证成功创建 5 分钟会话 |
| 危险操作拦截 | 未验证时拦截，验证后放行 |
| 只读模式绕过 | OTP 命令在只读模式下可正常执行 |
| 二维码格式 | 扫描后显示正确账号和 issuer（灵畿CLI） |

### 4.2 性能验收

| 验收项 | 标准 | 实际值 |
|--------|------|--------|
| OTP 生成响应时间 | < 100ms | ~10ms |
| OTP 验证响应时间 | < 100ms | ~10ms |
| 二维码生成时间 | < 100ms | ~5ms |

### 4.3 安全验收

| 验收项 | 标准 |
|--------|------|
| 密钥随机性 | 使用 crypto/rand 生成 |
| 密钥存储 | 仅本地存储，不上传 |
| 时序攻击防护 | 使用常量时间比较 |
| 关闭保护 | 关闭 OTP 需要 OTP 验证 |
| 只读绕过 | OTP 命令绕过只读检查 |
