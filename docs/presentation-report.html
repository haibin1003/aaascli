AI 拥有强大的信息整合、逻辑推理和代码生成能力，但缺乏直接访问企业内部平台的通道，导致「有脑子、没手脚」，最终沦为一个只能给建议的聊天工具。

🛡️

关键优势：对原有系统零侵入

本项目无需修改平台后端一行代码、无需新增接口、无需调整数据库或网络策略。所有能力均通过纯客户端方式（命令行 + 浏览器插件）实现，对现有山东能力开放平台完全透明，风险可控，即插即用。

💡

核心解决思路

为 AI 助手打造一款专用的命令行客户端（CLI），通过深入研究平台前端加密与通信机制，完整还原 RSA+AES 混合加密协议，让 AI 能够直接调用平台真实接口。AI 从此可以查询能力目录、检索 API 服务、查看接口详情，并基于返回数据主动生成调用代码和完整业务方案。

三大技术突破

🛡️ RSA+AES 混合加密还原

平台前端采用 RSA-2048 + AES-128-CBC-PKCS7 混合加密。我们通过逆向前端加密逻辑，在 Go 中完整实现了密钥生成、请求加密、响应解密的全流程，确保 CLI 与浏览器请求行为完全一致。

🔤 GBK 编码自动兼容

部分平台接口返回 GBK 编码的 JSON 数据，直接解析会出现中文乱码。我们内置了 GBK 到 UTF-8 的自动检测与转码机制，并将 golang.org/x/text 完整 Vendor 到本地，确保离线环境可用。

🍪 多 Cookie 联合认证

平台认证不仅依赖 Token，还需要 openPortalVerificationCode 和 openPortalServiceID 两个辅助 Cookie。我们通过 Chrome 扩展完整提取三 Cookie，并在 CLI 中自动注入到每个请求的 Header 和 Cookie 中。

03 / 核心功能

五大模块，覆盖能力查询、服务检索、知识赋能和智能辅助

📦

能力管理（Ability）

  • 查询全部 325+ 业务能力列表（支持分页）
  • 按关键词快速搜索能力（名称/编码/分类）
  • 查看能力详情（描述、提供方、分类、调用类型）
  • 浏览能力下挂载的服务菜单
  • 查询我的已订购能力
🔌

数字服务管理（Service）

  • 查询全量 3800+ 服务目录（目录+API 混合树）
  • 搜索具体 API 服务接口
  • 查看服务详情：URL、方法、协议、版本
  • 获取请求/响应示例 JSON
  • 查看接口负责人、部门、联系方式
📚

知识库（Knowledge）

  • 内置能力使用指南（Markdown 文档）
  • 典型业务场景和组合方案
  • Python / Java / Go / cURL 代码模板
  • 常见问题与故障排查
  • 支持全文搜索和上下文片段提取
🔐

认证与辅助（Auth & Helper）

  • Chrome 插件一键提取 Token
  • CLI 自动保存登录凭证到本地
  • TLS 跳过验证，适配内网环境
  • AI onboarding 引导指南（sdp onboard）
  • 格式化 JSON 输出，便于阅读

06 / 成效与价值

我们做这件事，究竟带来了什么实质性的改变？

🚀 AI 从「顾问」变成「执行者」

过去 AI 只能告诉用户「你可以去平台搜一下这个能力」。现在 AI 可以直接执行查询、分析返回结果、输出完整方案，真正实现了「动手」而不仅是「动嘴」，AI 的价值边界大幅拓展。

核心价值

⏱️ 能力匹配效率提升 10 倍以上

平台 325+ 能力、3800+ 服务全靠人工浏览需要 1-2 小时。通过 CLI + AI 检索，用户只需描述业务场景，30 秒内即可获得精准的能力推荐、接口详情和示例代码。

效率提升

🛠️ 显著降低技术接入门槛

AI 可基于接口返回的 requestExample 和 responseExample，自动生成多语言调用代码。普通业务人员无需阅读复杂文档，复制代码稍作修改即可完成对接，开发周期大幅缩短。

降本增效

🌐 一次构建，全平台覆盖

同时提供 Windows(x64/ARM64)、Linux(x64/ARM64)、macOS(Intel/Apple Silicon) 六个版本的二进制文件。无论 AI 运行在本地电脑、云服务器还是容器环境中，都能即装即用。

工程优势

📚 知识沉淀，越用越聪明

内置知识库将短信触达、机器视觉、定位服务等典型方案固化到二进制中。AI 可以直接引用最佳实践，而不是每次从零推理，回答质量更稳定，新人上手更快。

知识资产

🛡️ 对原有系统零侵入

无需平台方改接口、无需动数据库、无需调整网络策略或权限配置。工具作为纯客户端独立运行，所有通信行为与正常浏览器访问完全一致，对现有平台架构完全透明，部署风险极低。

部署优势

🔒 安全合规，完全离线可用

所有 Go 依赖均已 Vendor 化，可在完全离线的内网环境编译和运行。Token 和 Cookie 仅保存在本地配置文件，不经过任何第三方中间服务，安全可控。

07 / 未来规划
