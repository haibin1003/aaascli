# LC CLI 功能开发指南

本文档指导 AI 如何为 LC CLI 工具开发新功能，基于从浏览器开发者工具获取的 API 信息生成完整的命令实现。

---

## 一、代码架构概览

### 1.1 项目结构

```
.
├── main.go                    # 入口文件
├── cmd/lc/                    # 命令定义
│   ├── root.go               # 根命令、全局标志
│   ├── repo.go               # 仓库命令 (示例)
│   ├── pr.go                 # PR/MR 命令 (示例)
│   ├── space.go              # 空间命令 (示例)
│   └── cookie.go             # Cookie 命令 (示例)
└── internal/
    ├── api/                   # API 客户端层
    │   ├── client.go         # HTTP 客户端
    │   ├── project.go        # 项目 API (示例)
    │   ├── merge_request.go  # MR API (示例)
    │   └── space.go          # 空间 API (示例)
    └── config/               # 配置管理
        └── config.go         # 配置结构和加载
```

### 1.2 核心设计模式

| 层级 | 职责 | 关键文件 |
|------|------|----------|
| **命令层** | Cobra 命令定义、参数解析、用户交互 | `cmd/lc/*.go` |
| **API 层** | HTTP 请求封装、请求/响应结构体 | `internal/api/*.go` |
| **配置层** | 配置加载、Header 生成、认证管理 | `internal/config/config.go` |

### 1.3 命令命名规范

```
lc <资源> <动作> [参数]

示例:
- lc repo list          # 列出仓库
- lc pr create          # 创建合并请求
- lc space list         # 列出空间
- lc repo delete [id]   # 删除仓库
```

## 二、从 API 到代码的工作流程

### 2.1 获取 API 信息

在浏览器开发者工具中：

1. **定位到目标功能**，打开 Network 面板
2. **执行操作**（如点击按钮）
3. **右键请求 → Copy → Copy as cURL (bash)**
4. **复制响应 JSON**（Response 标签页）

### 2.2 分析 API 信息

#### 2.2.1 提取关键信息

从 cURL 中提取：

```bash
curl 'https://rdcloud.4c.hq.cmcc/moss/web/.../api/endpoint' \
  -X 'POST' \
  -H 'Content-Type: application/json' \
  -H 'X-Auth-AppId: CMDEVOPS' \
  -H 'X-Auth-ModuleId: XXJSxiaobaice' \
  --data-raw '{"key":"value"}'
```

| 信息 | 示例 | 用途 |
|------|------|------|
| HTTP 方法 | POST | API 层 Method 字段 |
| URL 路径 | /api/endpoint | API 层 URL 构建 |
| Path 参数 | /projects/123 | 命令参数设计 |
| Query 参数 | ?page=1&size=20 | 命令 flags 设计 |
| 请求体字段 | `{"name":"xxx"}` | Request 结构体 |
| 响应字段 | `{"id":123}` | Response 结构体 |
| Header 中的变量 | X-Auth-ModuleId | 工作空间参数 |

#### 2.2.2 识别资源类型

根据 URL 确定命令分类：

| URL 模式 | 资源类型 | 命令前缀 |
|----------|----------|----------|
| `/projects/...` | 仓库/项目 | `lc repo` |
| `/merge-requests/...` | 合并请求 | `lc pr` |
| `/spaces/...` | 研发空间 | `lc space` |
| `/requirements/...` | 需求 | `lc req` |
| `/tasks/...` | 任务 | `lc task` |

## 三、代码生成指南

### 3.1 提示词模板

将以下信息提供给 AI：

````markdown
## 任务
为 LC CLI 添加新命令：`lc <资源> <动作>`

## 输入信息

### cURL 请求
```bash
[Paste cURL here]
```

### 响应示例
```json
[Paste JSON response here]
```

### 命令设计
- 命令格式：`lc <resource> <action> [args]`
- 必需参数：[列出参数]
- 可选参数：[列出 flags]

## 要求

1. **命令层** (`cmd/lc/<resource>.go`):
   - 使用 Cobra 框架
   - 支持 `-w/--workspace` flag（如需要工作空间）
   - 支持全局 `-d/--debug` 和 `-k/--insecure` flags
   - 必需参数使用 `Args: cobra.ExactArgs(n)` 或 `MarkFlagRequired()`
   - 输出必须是格式化的 JSON

2. **API 层** (`internal/api/<resource>.go`):
   - 定义 Request/Response 结构体
   - 创建 Service 结构体和方法
   - 使用已有的 `Client` 发送 HTTP 请求
   - URL 使用 `cfg.API.BaseRepoURL` 或 `cfg.API.BasePlatformURL` 作为前缀

3. **配置使用**:
   - 使用 `config.NewConfig()` 获取默认配置
   - 使用 `config.LoadConfigWithDefaults()` 加载用户配置
   - 使用 `cfg.GetHeadersWithWorkspace(workspaceKey)` 获取请求头

4. **HTTP 客户端**:
   - 根据 `debugMode` 创建客户端：`api.NewClientWithLogger(logger)` 或 `api.NewClient()`
   - 根据 `insecureSkipVerify` 创建安全/非安全客户端

5. **输出格式**:
   ```json
   {
     "success": true,
     "data": { ... },
     "message": "操作成功"
   }
   ```

6. **错误处理**:
   - API 错误：`fmt.Printf("Error: %s\n", resp.Message)` + `os.Exit(1)`
   - HTTP 错误：`fmt.Printf("Error: %v\n", err)` + `os.Exit(1)`
   - 解析错误：`fmt.Printf("Error parsing response: %v\n", err)` + `os.Exit(1)`

## 参考代码

参考 `cmd/lc/repo.go` 中的 `listRepositories()` 函数和 `internal/api/project.go` 中的 `ListUserRepos()` 方法。
````

### 3.2 AI 处理步骤

AI 应该按以下步骤生成代码：

#### 步骤 1：分析 API 确定结构

```
分析 cURL:
├── HTTP 方法 → 确定 Service 方法逻辑
├── URL 路径 → 确定 API endpoint
├── Path 参数 → 确定命令参数
├── Query 参数 → 确定命令 flags
├── 请求体 → 定义 Request 结构体
└── 响应体 → 定义 Response 结构体
```

#### 步骤 2：生成 API 层代码

创建/修改 `internal/api/<resource>.go`：

```go
// 1. 定义请求结构体
type XxxCreateRequest struct {
    Name string `json:"name"`  // 根据 cURL body 定义
    // ... 其他字段
}

// 2. 定义响应结构体
type XxxCreateResponse struct {
    Success bool   `json:"success"`
    Code    string `json:"code"`
    Message string `json:"message"`
    Data    XxxData `json:"data"`
}

type XxxData struct {
    Id   int    `json:"id"`
    Name string `json:"name"`
    // ... 根据响应 JSON 定义
}

// 3. 定义 Service
type XxxService struct {
    BaseURL    string
    Headers    map[string]string
    HTTPClient HTTPClient
}

func NewXxxService(baseURL string, headers map[string]string, client HTTPClient) *XxxService {
    return &XxxService{BaseURL: baseURL, Headers: headers, HTTPClient: client}
}

// 4. 实现 API 方法
func (s *XxxService) Create(req *XxxCreateRequest) (*XxxCreateResponse, error) {
    apiReq := &Request{
        URL:     s.BaseURL + "/xxx/create",
        Method:  "POST",
        Headers: s.Headers,
        Body:    req,
    }

    resp, err := s.HTTPClient.Send(apiReq)
    if err != nil {
        return nil, err
    }
    defer resp.Body.Close()

    body, _ := io.ReadAll(resp.Body)
    var result XxxCreateResponse
    if err := json.Unmarshal(body, &result); err != nil {
        return nil, fmt.Errorf("failed to parse: %v", err)
    }
    return &result, nil
}
```

#### 步骤 3：生成命令层代码

创建/修改 `cmd/lc/<resource>.go`：

```go
package cmd

import (
    "encoding/json"
    "fmt"
    "os"

    "github.com/spf13/cobra"
    "go.uber.org/zap"
    "github.com/user/lc/internal/api"
    "github.com/user/lc/internal/config"
)

// 1. 定义 flag 变量
var (
    xxxWorkspaceKey string
    xxxName         string
    xxxId           int
)

// 2. 定义父命令
var xxxCmd = &cobra.Command{
    Use:   "xxx",
    Short: "管理XXX资源",
    Long:  `管理XXX资源，包括创建、查询等功能。`,
}

// 3. 定义子命令
var xxxCreateCmd = &cobra.Command{
    Use:   "create",
    Short: "创建XXX",
    Long:  `创建一个新的XXX资源。`,
    Run: func(cmd *cobra.Command, args []string) {
        createXxx()
    },
}

var xxxListCmd = &cobra.Command{
    Use:   "list",
    Short: "列出XXX",
    Long:  `列出XXX资源列表。`,
    Run: func(cmd *cobra.Command, args []string) {
        listXxx()
    },
}

// 4. 注册命令和 flags
func init() {
    rootCmd.AddCommand(xxxCmd)
    xxxCmd.AddCommand(xxxCreateCmd)
    xxxCmd.AddCommand(xxxListCmd)

    // 必需 flags
    xxxCreateCmd.Flags().StringVarP(&xxxWorkspaceKey, "workspace", "w", "", "研发空间 key (必填)")
    xxxCreateCmd.Flags().StringVarP(&xxxName, "name", "n", "", "名称 (必填)")
    xxxCreateCmd.MarkFlagRequired("workspace")
    xxxCreateCmd.MarkFlagRequired("name")

    xxxListCmd.Flags().StringVarP(&xxxWorkspaceKey, "workspace", "w", "", "研发空间 key (必填)")
    xxxListCmd.MarkFlagRequired("workspace")
}

// 5. 实现命令逻辑
func createXxx() {
    // 5.1 初始化 logger (debug 模式)
    if debugMode {
        logger, _ = zap.NewDevelopment()
        defer logger.Sync()
    }

    // 5.2 加载配置
    cfg := config.NewConfig()
    if loadedCfg, err := config.LoadConfigWithDefaults(config.GetDefaultConfigPath()); err == nil {
        cfg = loadedCfg
    }

    // 5.3 创建 HTTP 客户端
    var httpClient *api.Client
    if debugMode {
        if insecureSkipVerify {
            httpClient = api.NewInsecureClientWithLogger(logger)
        } else {
            httpClient = api.NewClientWithLogger(logger)
        }
    } else {
        if insecureSkipVerify {
            httpClient = api.NewInsecureClient()
        } else {
            httpClient = api.NewClient()
        }
    }

    // 5.4 创建 Service 并调用 API
    headers := cfg.GetHeadersWithWorkspace(xxxWorkspaceKey)
    xxxService := api.NewXxxService(cfg.API.BaseRepoURL, headers, httpClient)

    requestData := &api.XxxCreateRequest{
        Name: xxxName,
        // ... 其他字段
    }

    resp, err := xxxService.Create(requestData)
    if err != nil {
        fmt.Printf("Error: %v\n", err)
        os.Exit(1)
    }

    // 5.5 检查响应并输出
    if !resp.Success {
        fmt.Printf("创建失败: %s\n", resp.Message)
        os.Exit(1)
    }

    // 5.6 格式化输出
    output := map[string]interface{}{
        "success": true,
        "data":    resp.Data,
        "message": "创建成功",
    }

    jsonData, _ := json.MarshalIndent(output, "", "  ")
    fmt.Println(string(jsonData))
}
```

## 四、代码规范检查清单

### 4.1 命令层检查项

| 检查项 | 正确示例 | 错误示例 |
|--------|----------|----------|
| 全局 flags 使用 | `if debugMode { logger, _ = zap.NewDevelopment() }` | 直接创建 logger |
| 配置加载 | `cfg := config.NewConfig()` + `LoadConfigWithDefaults()` | 直接读取文件 |
| HTTP 客户端 | 根据 debugMode/insecure 选择 | 只使用一种客户端 |
| Header 获取 | `cfg.GetHeadersWithWorkspace(key)` | 手动构建 headers |
| 错误处理 | `fmt.Printf()` + `os.Exit(1)` | 返回 error |
| 输出格式 | JSON，使用 `json.MarshalIndent` | 纯文本、fmt.Printf 直接输出 |
| 必需参数 | `MarkFlagRequired()` 或 `cobra.ExactArgs()` | 代码中手动检查 |

### 4.2 API 层检查项

| 检查项 | 正确示例 | 错误示例 |
|--------|----------|----------|
| URL 构建 | `fmt.Sprintf("%s/projects/%d", s.BaseURL, id)` | 硬编码完整 URL |
| 响应解析 | 使用结构体 `json.Unmarshal(body, &result)` | 直接操作 map |
| 资源关闭 | `defer resp.Body.Close()` | 不关闭 body |
| 错误返回 | `return nil, fmt.Errorf("...: %w", err)` | `return nil, err` |

### 4.3 JSON 字段规范

```go
// 使用 camelCase 与 API 保持一致
type Response struct {
    Success     bool   `json:"success"`     // 小写开头
    Code        string `json:"code"`
    Message     string `json:"message"`
    GitProjectId int   `json:"gitProjectId"` // ID 缩写大写
    HttpPath    string `json:"httpPath"`     // HTTP 缩写大写
}
```

## 五、常见模式参考

### 5.1 GET 请求（列表查询）

```go
// API 层
func (s *XxxService) List(spaceCode string, page, size int) (*http.Response, error) {
    url := fmt.Sprintf("%s/xxx?page=%d&size=%d", s.BaseURL, page, size)
    apiReq := &Request{URL: url, Method: "GET", Headers: s.Headers}
    return s.HTTPClient.Send(apiReq)
}

// 命令层 - 需要手动解析响应体
resp, err := service.List(workspace, page, size)
body, _ := io.ReadAll(resp.Body)
var result struct {
    Success bool `json:"success"`
    Data    []XxxItem `json:"data"`
}
json.Unmarshal(body, &result)
```

## 六、调试与测试

### 6.1 使用 Debug 模式

```bash
# 启用 debug 查看详细日志
./lc repo list -w XXJSxiaobaice -d

# 跳过 SSL 验证（开发环境）
./lc repo list -w XXJSxiaobaice -k
```

### 6.2 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| nil pointer panic | logger.Debug() 在 debugMode=false 时调用 | 包装在 `if debugMode {}` 中 |
| json unmarshal 错误 | 结构体字段类型与响应不匹配 | 检查 JSON 数字/字符串类型 |
| 401 未授权 | Cookie 失效 | 运行 `lc cookie set <新cookie>` |
| 404 未找到 | URL 路径错误 | 检查 baseURL 和 endpoint 拼接 |

### 6.3 验证步骤

1. 编译: `go build -o lc .`
2. 帮助: `./lc <resource> --help`
3. 子命令帮助: `./lc <resource> <action> --help`
4. 执行: `./lc <resource> <action> [args]`
5. 验证输出: 必须是合法 JSON

### 8.2 关键类型定义

```go
// HTTPClient 接口 - api/client.go
type HTTPClient interface {
    Send(*Request) (*http.Response, error)
}

// Client - 实际 HTTP 客户端
type Client struct {
    HTTPClient *http.Client
    Logger     *zap.Logger
}

// Request - API 请求封装
type Request struct {
    URL     string
    Method  string
    Headers map[string]string
    Body    interface{}
}
```
