## 模块详解

### 1. 命令层 (cmd/lc/)

使用 [Cobra](https://github.com/spf13/cobra) 框架实现命令行接口。

#### 核心文件

| 文件 | 职责 | 关键结构 |
|------|------|----------|
| `root.go` | 根命令定义、全局标志 | `rootCmd`, `versionCmd` |
| `req.go` | 需求管理命令 | `reqCmd`, `reqCreateCmd` |
| `bug.go` | 缺陷管理命令 | `bugCmd`, `bugCreateCmd` |
| `pr.go` | 合并请求命令 | `prCmd`, `prCreateCmd` |
| `repo.go` | 仓库管理命令 | `repoCmd`, `repoCreateCmd` |
| `space.go` | 研发空间命令 | `spaceCmd`, `spaceProjectCmd` |
| `task.go` | 任务管理命令 | `taskCmd`, `taskCreateCmd` |
| `detect.go` | 自动探测命令 | `detectCmd` |
| `skills.go` | AI Skill 命令 | `skillsCmd`, `skillsShowCmd` |

#### 命令实现模式

```go
var xxxCmd = &cobra.Command{
    Use:   "create",
    Short: "简短描述",
    Long:  `详细描述`,
    Run: func(cmd *cobra.Command, args []string) {
        // 1. 获取参数
        workspaceKey, _ := cmd.Flags().GetString("workspace-key")

        // 2. 尝试自动探测
        autoResult := common.TryAutoDetect(true)
        if autoResult.Success && workspaceKey == "" {
            workspaceKey = autoResult.Context.WorkspaceKey
        }

        // 3. 使用 Execute 执行命令
        common.Execute(func(ctx *common.CommandContext) (interface{}, error) {
            // 4. 调用 API
            result, err := api.CreateSomething(ctx, workspaceKey, ...)
            return result, err
        }, common.ExecuteOptions{
            DebugMode: debugMode,
            Insecure:  insecureSkipVerify,
        })
    },
}
```

### 2. 通用层 (internal/common/)

提供命令执行的通用基础设施。

#### 2.1 CommandContext (context.go)

命令执行的上下文容器，统一管理依赖：

```go
type CommandContext struct {
    Config    *config.Config      // 配置信息
    Client    *api.Client         // HTTP 客户端
    Logger    *zap.Logger         // 日志器
    DebugMode bool                // 调试模式
    Insecure  bool                // 跳过 SSL 验证
    DryRun    bool                // 模拟执行
}
```

**使用方式**：

```go
ctx, err := common.NewCommandContext(debugMode, insecure, dryRun, cookie)
if err != nil {
    return err
}
defer ctx.Close()

// 获取 HTTP 头
headers := ctx.GetHeaders(workspaceKey)
```

#### 2.2 Executor (executor.go)

统一的命令执行器，负责：
- 初始化 CommandContext
- 执行命令函数
- 统一输出格式（JSON）
- 错误处理

**核心函数**：

```go
// Execute - 标准执行，自动包装为统一 JSON 格式
func Execute(fn CommandFunc, opts ExecuteOptions)

// ExecuteWithOutput - 自定义输出（用于非 JSON 输出）
func ExecuteWithOutput(fn CommandFuncWithOutput, opts ExecuteOptions)

// PrintJSON - 打印格式化 JSON
func PrintJSON(v interface{})

// PrintError - 打印错误信息
func PrintError(err error)
```

**统一输出格式**：

```go
type CommandResult struct {
    Success bool        `json:"success"`
    Data    interface{} `json:"data,omitempty"`
    Error   interface{} `json:"error,omitempty"`
    Meta    MetaInfo    `json:"meta"`
}

type MetaInfo struct {
    Timestamp     string `json:"timestamp"`
    ExecutionTime int64  `json:"executionTime"`
    Version       string `json:"version"`
}
```

#### 2.3 AutoDetect (autodetect.go)

Git 仓库自动探测功能：

```go
type AutoDetectContext struct {
    WorkspaceKey   string                 // 研发空间 Key
    WorkspaceName  string                 // 研发空间名称
    TenantID       string                 // 租户 ID
    GitProjectID   string                 // Git 项目 ID
    Repository     map[string]interface{} // 仓库详情
    Matched        bool                   // 是否匹配成功
}

// 尝试自动探测
func TryAutoDetect(requireGitRepo bool) *AutoDetectResult

// 获取探测上下文（带缓存）
func GetAutoDetectContext() (*AutoDetectContext, error)
```

## 数据流

### 命令执行流程

```
用户输入 → Cobra 解析 → 参数处理 → 自动探测 → API 调用 → 统一输出
```

详细步骤：

1. **解析阶段** (Cobra)
   - 解析命令、子命令、标志
   - 验证必需参数

2. **准备阶段** (Command)
   - 获取显式参数
   - 尝试自动探测缺失参数
   - 验证参数完整性

3. **执行阶段** (Executor)
   - 创建 CommandContext
   - 初始化 Client、Config、Logger
   - 调用 API

4. **输出阶段** (Executor)
   - 包装结果为统一格式
   - 添加 Meta 信息
   - 输出 JSON

## 关键设计模式

### 1. 命令模式 (Command Pattern)

每个命令都是一个 `cobra.Command` 实例，封装了：
- 命令元数据（名称、描述、用法）
- 参数解析
- 执行逻辑

### 2. 依赖注入 (Dependency Injection)

通过 `CommandContext` 统一管理依赖：

```go
func Execute(fn CommandFunc, opts ExecuteOptions) {
    ctx, err := NewCommandContext(opts.DebugMode, opts.Insecure, opts.DryRun, opts.Cookie)
    // ...
    result, err := fn(ctx)  // 注入 ctx
}
```

### 3. 模板方法模式 (Template Method)

命令执行遵循固定模板：

```go
Run: func(cmd *cobra.Command, args []string) {
    // 1. 获取参数
    // 2. 自动探测
    // 3. Execute 执行
}
```

### 4. 缓存模式 (Cache Pattern)

自动探测结果缓存：

```go
var (
    cachedDetectResult *AutoDetectContext
    cachedDetectMu     sync.RWMutex
    cacheValidDuration = 5 * time.Minute
)
```

## 扩展点

### 添加新命令

1. 在 `cmd/lc/` 创建新文件（如 `feature.go`）
2. 定义 `cobra.Command` 结构
3. 在 `init()` 中注册到 rootCmd
4. 实现业务逻辑（调用 `internal/api/`）

参见：[05-新增命令指南.md](./05-新增命令指南.md)

### 添加新 API

1. 在 `internal/api/` 添加方法
2. 使用 `Client.Send()` 发送 HTTP 请求
3. 处理响应并返回结构化数据

参见：[06-API开发规范.md](./06-API开发规范.md)

## 依赖关系

```
cmd/lc/*
    ↓
internal/common/*  ←→  internal/config/*
    ↓                    ↓
internal/api/* ←───────┘
```

**依赖原则**：
- 上层可以依赖下层
- 同层可以互相依赖
- 避免循环依赖
- 禁止下层依赖上层
